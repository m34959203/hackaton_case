# Методология анализа

## Обзор

ZanAlytics использует комбинацию методов NLP, машинного обучения и больших языковых моделей (LLM) для автоматического выявления проблем в законодательстве РК. Система не заменяет юридического эксперта, а помогает ему, сокращая объём ручной работы и выделяя нормы, требующие внимания.

## 1. Сбор данных

### Источник
Официальная информационно-правовая система **Әділет** ([adilet.zan.kz](https://adilet.zan.kz)), администрируемая Институтом законодательства и правовой информации при Министерстве юстиции РК.

### Извлечение
Для каждого документа извлекаются:
- **Текст** — полный текст НПА со структурой (части, главы, статьи, пункты)
- **Метаданные** — дата принятия, дата последнего изменения, юридическая сила, принявший орган, сфера правоотношений, статус (действующий/утративший силу)
- **Перекрёстные ссылки** — какие документы ссылаются на данный и на какие ссылается он

### Структурный парсинг
Текст каждого документа разбивается на иерархию:
```
Документ → Часть → Глава → Статья → Пункт (Норма)
```
Каждая норма (пункт статьи) является минимальной единицей анализа.

## 2. Векторное представление (Embedding)

### Модель
Используется **BAAI/bge-m3** — мультиязычная модель эмбеддингов (1024 измерения), обученная на текстах включая русский язык.

### Процесс
1. Каждая норма преобразуется в вектор фиксированной размерности (1024)
2. Вектор отражает **семантический смысл** нормы, а не просто наличие ключевых слов
3. Векторы сохраняются в ChromaDB для быстрого поиска по сходству

### Почему не ключевые слова?
Две нормы могут описывать одно и то же разными словами. Семантические эмбеддинги улавливают смысловое сходство:
- «Срок рассмотрения заявления составляет 30 дней»
- «Заявка подлежит рассмотрению в тридцатидневный срок»
→ Cosine similarity ≈ 0.92 (высокое сходство)

## 3. Кластеризация

### Алгоритм
**UMAP** (Uniform Manifold Approximation and Projection) для снижения размерности + **HDBSCAN** (Hierarchical Density-Based Spatial Clustering) для группировки.

### Зачем?
Сравнивать каждую пару норм из 5000 (≈12.5 млн пар) вычислительно невозможно. Кластеризация группирует нормы по тематике, и анализ проводится только внутри тематических групп.

### Результат
Каждая норма получает:
- `cluster_id` — номер тематического кластера
- `topic_label` — тематика (генерируется LLM по центральным нормам кластера)

## 4. Выявление проблем

### 4.1. Дублирование

**Критерий**: cosine similarity между двумя нормами > 0.85 внутри одного кластера.

**Подтверждение**: LLM проверяет, действительно ли нормы говорят одно и то же. Это исключает ложные срабатывания на нормы, которые похожи по тексту, но регулируют разные аспекты.

**Пример**:
- Закон А, ст.15: «Работодатель обязан обеспечить безопасные условия труда»
- Закон Б, ст.22: «Обеспечение безопасных условий труда — обязанность работодателя»
→ Дублирование (cosine similarity = 0.91, подтверждено LLM)

### 4.2. Противоречие

**Критерий**: нормы в одном кластере (тематически связаны), но с различающимися предписаниями.

**Метод**: LLM-as-Judge — языковая модель анализирует две нормы и определяет:
1. Тип соотношения: ПРОТИВОРЕЧИЕ / СОГЛАСОВАНЫ / НЕОПРЕДЕЛЁННО
2. Уровень уверенности: высокий / средний / низкий
3. Конкретные противоречащие фрагменты
4. Рекомендацию по устранению

**Юридический контекст в промптах**:
- **Lex posterior** — более поздний закон имеет приоритет
- **Lex specialis** — специальный закон преобладает над общим
- **Иерархия НПА** — Конституция > Кодекс > Закон > Подзаконный акт

**Пример**:
- Закон А (2019), ст.15: «Срок рассмотрения составляет 30 дней»
- Закон Б (2022), ст.22: «Рассматривается в течение 15 рабочих дней»
→ Противоречие (разные сроки для одной процедуры)

### 4.3. Устаревшие нормы

**Критерии (комбинация)**:
1. **Статус документа** — пометка «утративший силу» на adilet.zan.kz
2. **Ссылки на отменённые акты** — норма ссылается на документ, который уже не действует
3. **Временные маркеры** — фразы «до 2020 года», «на период 2018-2020»
4. **LLM-проверка** — модель оценивает актуальность нормы в контексте текущего законодательства

## 5. Визуализация связей

### Граф
- **Узлы** — документы (крупные) и статьи (мелкие)
- **Рёбра** — перекрёстные ссылки (серые), противоречия (красные), дублирования (оранжевые)
- **Размер узла** — пропорционален количеству связей
- **Цвет узла** — по типу документа (кодекс, закон, указ, постановление)

### Метрики графа
- **PageRank** — выявляет наиболее «влиятельные» документы
- **Betweenness centrality** — документы-мосты между правовыми областями
- **Community detection** — автоматическое выделение правовых кластеров

## 6. Объяснимость (Explainable AI)

Для каждого обнаружения система предоставляет:

1. **Текст обеих норм** с подсветкой конфликтующих фрагментов
2. **Объяснение LLM** — почему нормы противоречат/дублируют друг друга
3. **Уровень уверенности** — высокий / средний / низкий
4. **Правовой контекст** — даты, иерархия, область применения
5. **Рекомендация** — что можно предпринять для устранения проблемы
6. **Кнопка «Неверное обнаружение»** — human-in-the-loop для обратной связи

## 7. Ограничения метода

| Ограничение | Описание | Митигация |
|-------------|----------|-----------|
| Ложноположительные | LLM может ошибочно классифицировать нормы как противоречащие | Уровень уверенности + верификация экспертом |
| Контекст | Система анализирует текст, но не правоприменительную практику | Явное указание в отчётах |
| Полнота | Демо-набор покрывает ~50 документов из 230 000+ | Архитектура масштабируема |
| Языковая модель | qwen2.5:14b не специализирована на юридических текстах | Специализированные промпты с юридическим контекстом |
| Казахский язык | Текущая версия работает с русскоязычными текстами | Модель bge-m3 поддерживает казахский, расширение запланировано |

## 8. Валидация

### Качественная
- Ручная проверка выборки обнаружений юридическим экспертом
- Оценка precision (доля истинных проблем среди обнаруженных)

### Количественная
- Сравнение с baseline: случайное назначение «противоречий»
- A/B сравнение: LLM-only vs. embedding+LLM pipeline
- Метрики: precision@k, recall (на размеченной выборке, если доступна)
