# Структура презентации

## Слайд 1: Титульный
- **ZanAlytics** — AI-система анализа законодательства РК
- Decentrathon 5.0 | Кейс 1: Законодательная энтропия
- Команда: Центр ИИ при ЖезУ

## Слайд 2: Проблема
- 230 000+ нормативных актов в базе Әділет
- Поправки накладываются на поправки → противоречия, дублирование
- Юристы тратят **тысячи часов** на проверку актуальности
- 2026 = Год цифровизации и ИИ, но **ноль AI-решений** для анализа законов в РК
- Рынок LegalTech AI: **$2.8 млрд** (2025), рост 31% в год

## Слайд 3: Решение
- ZanAlytics автоматически:
  1. Парсит НПА с adilet.zan.kz
  2. Выявляет **противоречия** между нормами
  3. Находит **дублирования** (одно и то же разными словами)
  4. Обнаруживает **устаревшие** нормы
  5. **Объясняет** каждое обнаружение (Explainable AI)

## Слайд 4: Демо — Дашборд
- Скриншот/видео дашборда
- Ключевые метрики: X документов, Y норм, Z проблем
- Распределение по типам и правовым областям

## Слайд 5: Демо — 3D-граф
- Скриншот/видео интерактивного графа
- Цветовая кодировка: кодексы (синий), законы (зелёный)
- Красные рёбра = противоречия, оранжевые = дублирование

## Слайд 6: Демо — Обнаружение противоречия
- Конкретный пример: две нормы бок-о-бок
- Подсветка конфликтующих фрагментов
- Объяснение ИИ: почему это противоречие
- Рекомендация по устранению

## Слайд 7: Как это работает (пайплайн)
- Схема: Парсинг → Эмбеддинги → Кластеризация → Анализ → Граф
- Ключевые технологии: bge-m3, HDBSCAN, qwen2.5:14b
- Explainability: LLM-as-Judge + подсветка фрагментов

## Слайд 8: Данные
- Источник: Әділет (adilet.zan.kz) — офиц. база НПА РК
- Для демо: ~50 документов в области трудового права
- Структурный парсинг: документ → статьи → пункты (нормы)
- ~5000 норм проанализировано

## Слайд 9: Архитектура
- Диаграмма: Frontend (Next.js) → Backend (FastAPI) → Ollama + ChromaDB + SQLite
- Docker Compose — запуск одной командой
- RTX 5090 (32 GB VRAM) — анализ на своём железе

## Слайд 10: Explainability
- Human-in-the-loop: ИИ не принимает решений, а **помогает** юристу
- Для каждого обнаружения:
  - Уровень уверенности (high/medium/low)
  - Текстовое объяснение
  - Ссылки на конкретные нормы
  - Кнопка «Неверное обнаружение»
- Принципы: lex posterior, lex specialis, иерархия НПА

## Слайд 11: Практическая польза
- **Для юристов госорганов**: сокращение времени аудита НПА
- **Для Минюста**: инструмент мониторинга правового поля
- **Для бизнеса**: снижение регуляторных рисков
- **Для граждан**: более ясное и непротиворечивое законодательство

## Слайд 12: Ограничения и дальнейшие шаги
### Ограничения
- Демо покрывает ~50 из 230 000+ документов
- LLM может ошибаться — нужна верификация юристом
- Только русский язык (казахский — в планах)

### Дальнейшие шаги
- Расширение на всю базу Әділет (230 000+ НПА)
- Поддержка казахского языка (KazLLM)
- Интеграция с eGov и системами Минюста
- Мониторинг новых НПА в реальном времени (RSS)
- Потенциал: **первый LegalTech AI-продукт в Казахстане**
